Metadata Matching

When saving your own song impressions on DiggerHub, the title/artist/album fields all need to match exactly for it to be considered the same song. An original studio release and the live version are not the same. When matching between fans, an exact match is preferable, but a soft match on title and artist is enough for default ratings or recomendations.

If the local song had been in a "Talking Heads" folder, then Digger would have filled out the metadata using the path. With no metadata and incomplete folder structure, there's no match. If you have files with missing metadata, you can use a tagging tool like MuiscBrainz Picard to fix them. The new metadata will be read when you next play the song.

If you have changed quite a few files, click "Read Files" in the Digger Library Actions to update Digger. It might take a few minutes for the changes to sync with DiggerHub, but after that you may find your fans have more to contribute, and you'll have more info for them.
